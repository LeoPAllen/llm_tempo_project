Purpose
This file is the working script/specification for the full oTree experiment. It is intended to be the easiest document for the research team to edit. The team can revise wording, ordering, measures, and treatment structure here, then pass the file back for code updates.

How to use this file
- Treat this file as the human-readable source of truth for participant-facing content and study structure.
- Participant-facing labels are intentionally neutral and do not explicitly reveal measurement intent.
- Implementation notes are included but should not be shown to participants.


================================================================================
EXPERIMENTAL DESIGN
================================================================================

Research question
How does the speed at which an AI streams its response (token-by-token display pace) affect trust in and reliance on AI advice?

Design
- Single-factor between-subjects
- Factor: streaming speed (fast vs slow)
- Participants are randomly assigned to one condition and it is held constant across all task rounds

Treatments
- fast_stream: 100ms start delay, 100ms per word (~2-3 seconds for a typical response)
- slow_stream: 100ms start delay, 300ms per word (~6-7 seconds for a typical response)

Key dependent variables
- Weight of Advice (WOA): |post_answer - pre_answer| / |llm_recommendation - pre_answer|
- Per-task mechanism measures: cognitive trust, affective trust, confidence in AI, post-confidence
- Pre-confidence (measured before seeing AI advice)

Paradigm
- Judge-Advisor System: participant gives initial answer, sees AI advice, then gives final answer
- 4 decision tasks in randomized order, same treatment across all tasks

Attention check question

Implementation notes

================================================================================
PAGES 5-8. TASK ROUNDS (repeated 4 times, randomized order)
================================================================================

App: tasks
4 rounds, one per task. Task order randomized per participant.

Current task pool (4 tasks)
1. Trolley Problem (trolley)
2. Dictator Game (dictator)
3. Time Preference (time_preference)
4. Gneezy-Potters Investment (gneezy_potters)

Each round has 4 pages:


--- PAGE A: INITIAL ANSWER (PreAnswerPage) ---

Participant-facing content
- Title: "Your Initial Answer"
- Shows task scenario
- Asks for numeric response + pre-confidence (7-point Likert)

Per-task details:

  TROLLEY PROBLEM
  - Scenario: Runaway trolley heading toward five people. Pulling switch redirects to track with one person.
  - Prompt: "How likely would you be to pull the switch?"
  - Range: 0 = definitely would not, 100 = definitely would
  - LLM recommendation target: 100

  DICTATOR GAME
  - Scenario: Given 100 USD to divide. Other person has no input and must accept.
  - Prompt: "How much of the 100 USD would you give to the other person?"
  - Range: 0 to 100
  - LLM recommendation target: 30

  TIME PREFERENCE
  - Scenario: Receive 75 USD today or wait 10 days for a larger amount. Both guaranteed.
  - Prompt: "What is the minimum amount you would need to receive in 10 days to prefer waiting over receiving 75 USD today?"
  - Range: 75 to 200
  - LLM recommendation target: 78

  GNEEZY-POTTERS INVESTMENT
  - Scenario: Given 100 USD. Can invest 0-100 in risky asset (50% chance of 2.5x, 50% chance of losing). Remainder kept. Example: investing 0 means keeping full 100 USD.
  - Prompt: "How much of the 100 USD would you invest in the risky asset?"
  - Range: 0 to 100
  - LLM recommendation target: 60


--- PAGE B: AI ADVICE (LlmAdvicePage) ---

Participant-facing content
- Instruction: "Click Ask AI to send the question below and see the AI's advice."
- Shows pre-filled prompt (not editable by participant)
- Ask AI button triggers streamed response at treatment speed
- Stop button available to interrupt streaming

Per-task AI prompts and outputs:

  TROLLEY PROBLEM
  - Prompt: "A runaway trolley is heading toward five people. I can pull a switch to divert it to another track where it would kill one person instead. What should I do?"
  - Output: "I recommend pulling the switch. Saving five lives at the cost of one is the better outcome."

  DICTATOR GAME
  - Prompt: "I have 100 USD to split between myself and another person. They must accept whatever I give. How much should I give?"
  - Output: "I recommend giving 30 USD. This balances fairness with self-interest, and is in line with what most people consider reasonable."

  TIME PREFERENCE
  - Prompt: "I can receive 75 USD today or wait 10 days for a larger amount. What is the minimum I should wait for?"
  - Output: "I recommend waiting for anything above 78 USD. Even a small premium is worth a 10-day wait since the payment is guaranteed and the effective return is very high."

  GNEEZY-POTTERS
  - Prompt: "I have 100 USD. I can invest any amount in a risky asset with a 50% chance of 2.5x return and a 50% chance of losing it. How much should I invest?"
  - Output: "I recommend investing 60 USD. The odds are in your favor, so investing more than half makes sense, but keeping some safe limits your risk."

Implementation notes
- Response is pre-loaded (not a live API call); streaming is simulated at treatment speed
- io_history captures full interaction log with timestamps
- interrupt_latency_submit and interrupt_latency_stream recorded if participant clicks Stop
- Next button enabled after streaming completes (or after 15s failsafe)


--- PAGE C: REVISED ANSWER (RevisedAnswerPage) ---

Participant-facing content
- Title: "Your Final Answer"
- Instruction: "Now that you have seen the AI's advice, you may update your answer or keep it the same."
- Shows task scenario again
- Same response prompt and range as initial answer

Implementation notes
- post_numeric_response recorded
- WOA computed in analysis: |post - pre| / |llm_recommendation - pre|


--- PAGE D: MECHANISM MEASURES (MechanismMeasuresPage) ---

Participant-facing content
- Instruction: "Thinking about your final decision and the AI's advice you just received, please rate the following statements."
- Items (all 7-point Likert):
  1. "I am confident in my final answer." (post_confidence)
  2. "The AI's recommendation was logical and well-reasoned." (cognitive_trust)
  3. "I felt comfortable following the AI's recommendation." (affective_trust)
  4. "The AI gave good advice on this task." (confidence_in_ai)

================================================================================
IMPLEMENTATION NOTES
================================================================================

Randomization
- Task order is fully randomized across the 4 tasks per participant
- Treatment (fast_stream / slow_stream) assigned once per participant via round-robin, held constant across all tasks

Session configurations
- "LLM Experiment (random assignment)" — for production/Prolific runs
- "LLM Experiment - Fast Stream" — forces fast_stream for coauthor testing
- "LLM Experiment - Slow Stream" — forces slow_stream for coauthor testing

Page timing
- All pages extend TimedPage, which records entry time and elapsed milliseconds
- Page times stored in participant.vars['page_times'] and player.page_times (JSON)

Gating logic (TimedPage.is_displayed)
- Consent declined -> all subsequent pages hidden
- Failed attention check -> all subsequent pages hidden (except FailedAttentionPage and Conclusion)

Data collected per task round
- task_id, treatment
- pre_numeric_response, post_numeric_response
- pre_confidence, post_confidence
- cognitive_trust, affective_trust, confidence_in_ai
- io_history (JSON: full interaction log with timestamps)
- interrupt_latency_submit, interrupt_latency_stream
- reflection_time
- interrupted_stream
- page_times (JSON)

Prolific integration
- Completion URL: configured via prolific_completion_url in session config
- Auto-redirect on conclusion page

Validation rules
- Trolley: 0-100
- Dictator: 0-100
- Time preference: 75-200
- Gneezy-Potters: 0-100
- Age: 18-90 (inline JS + server-side)

Changes from previous version (2025 -> 2026-03-19)
- Reduced from 6 tasks to 4 (removed Ultimatum Game, NPV, Asset Allocation)
- Simplified from 2x2 design (start delay x stream speed) to single-factor (stream speed only)
- Dropped start delay manipulation due to confound with perceived deliberation
- Reduced slow stream from 500ms to 300ms per word for ecological validity
- Moved trust/NFC/prior AI measures from pre-task to post-task survey
- Added manipulation check page (3 perceived speed items)
- Added attention check filtering (failed practice -> blocked)
- Added employment status and income range to demographics
- Added Prolific auto-redirect on conclusion
- Cleaned up all participant-facing text for consistency (AI terminology, removed redundant titles)
